Lightweight starter repository for the MB sync project. Use this space to capture context, goals, and day-to-day development details as code is added.

## Project Goals
- Document the systems and data that need to be synchronized.
- Capture success criteria (latency, throughput, correctness, security).
- Track open questions, risks, and constraints as they emerge.

## Getting Started
1. Clone the repo: `git clone https://github.com/adhithyaraiml2022-arch/ashwin-azer-mb-sync.git`
2. Create a feature branch: `git checkout -b <feature-name>`
3. Add the stack-specific setup (dependencies, environment files) required for your implementation.
4. Update this README with concrete setup steps once the stack is chosen.
